# 制作アプリのタイトル：ラノベならべ

## 制作アプリの説明
- ライトノベルをタイトルで検索して探せます
- その中から自分の好きな作品をお気に入りに保存できます
- 保存した作品に感想やコメントを残すことができます
- <img width="3240" height="2160" alt="Image" src="https://github.com/user-attachments/assets/7484e714-0d2b-4a00-b6da-a420f726c09a" />
- 検索結果が出たときの画面です カーソルオンで色が付きます
- <img width="3240" height="2160" alt="Image" src="https://github.com/user-attachments/assets/0ab135b5-89bc-46ae-9dbd-b11270260661" />
- お気に入りに保存したいものをドラッグドロップで移動して保存できます
- <img width="3240" height="2160" alt="Image" src="https://github.com/user-attachments/assets/b59aec79-836c-43ab-a78b-bcd5e178ac10" />
- お気に入り一覧をクリックすると詳細画面が開き、コメント追加・削除、お気に入りから外すなどの操作ができます

## 工夫した点・こだわった点
- FirebaseのAPIキーと楽天ブックスのAPIキーを秘匿して作成しました
- 調べるがまま実装していったら、自分でAPIを作ったことになってました
- 正直あまり深いところでは理解できてませんが・・・
- Firebaseと何かのAPIと組み合わせて動かしたいと思っていたのでうまくいってよかったです
- 見た目をきれいにはあまりできなかったかもしれませんが、色を変えたり、検索中などの待ち時間があるところでボタンに変化をつけたり、保存もドラッグドロップで動きをつけれたのはよかったです

## 次回トライしたこと（または機能）
- 楽天ブックスAPIの検索結果はジャンル指定のところがばらけてて不快だったので、国立国会図書館の検索結果をJSONで出力したものとisbnを使ってデータを保管したりしてみたいです
- ものすごい量のJSONデータになるので現実的なのかわかりませんが
- 大きいデータや集計など行えたら面白いことができそうなのでやってみたいです

## 備考（感想、シェアしたいこと等なんでも）
- 最初使ってたAPIがJSONじゃなくてxmlデータで、使える形に落とし込むのにめちゃくちゃ苦労した挙句情報量が足りなくて使うのやめたり、外部JSONデータダウンロードして照合してみようとして、JSON文字列が改行していっぱいつながってるタイプのデータでそのまま使えないやつで試行錯誤した結果、API関係ないじゃんってなって使わなくなったり・・・
- 今回は完成版に残ってない苦労が多数ありましたが、成長にはつながりました
- 苦労した残骸をコメントアウトして残したままにしてあるので、そのうち要点をまとめて備忘録にしようと思います（もしくは同じようなことがあったとき資料にします）
